1//! 自请求到响应的可插拔流水线：由若干 [`PipelineStage`] 顺序组成，**无节点或 `None` 时与未接 pipeline 行为一致**。
2//!
3//! 扩展方式：在独立 crate 中 `impl PipelineStage for YourType`，用 [`PipelineBuilder`] 组装后交给 [`crate::registry::ProviderRegistry::set_pipeline`]。
4//!
5//! 约定：`before_generate` 按注册顺序执行；`after_generate` / `after_generate_stream` 按**相同顺序**执行（非洋葱逆序），便于推理。
6//!
7//! # 执行模型（与「多节点业务链」的对照）
8//!
9//! [`crate::registry::ProviderRegistry`] 在挂有非空 pipeline 时，对单次 `generate` / `generate_stream` 的顺序为：
10//!
11//! 1. 克隆请求 → [`PipelineContext`]
12//! 2. 按注册顺序依次调用各阶段的 [`PipelineStage::before_generate`]
13//! 3. **恰好一次**对 [`crate::core::provider::LlmProvider`] 的调用（内部仍带重试）
14//! 4. 将结果写入 `ctx.response`
15//! 5. 按注册顺序依次调用各阶段的 [`PipelineStage::after_generate`]（流式为结束后 [`PipelineStage::after_generate_stream`]）
16//!
17//! 文本化数据流（箭头表示顺序，与一次 provider 调用的关系）：
18//!
19//! ```text
20//! Request
21//!   -> before[stage0] -> before[stage1] -> ... -> before[stageN]
22//!   -> LlmProvider.generate(请求为 ctx.request)
23//!   -> after[stage0] -> after[stage1] -> ... -> after[stageN]
24//!   -> Response
25//! ```
26//!
27//! **含义**：框架只提供「围绕**一次** provider 往返」的前后链；**没有**「每个节点自动对应一次独立 LLM 调用」的内置语义。
28//!
29//! # 典型业务节点如何落位
30//!
31//! | 概念（例：输入→记忆→意图→执行→输出） | 是否天然「一节点一框架钩子」 | 常见落位 |
32//! |--------------------------------------|------------------------------|----------|
33//! | 输入 | 是 | 初始 [`GenerateRequest`] / [`PipelineContext::request`] |
34//! | 记忆召回 | 部分 | [`PipelineStage::before_generate`] 内改 `messages` / `metadata`，或节点内调检索；若召回本身还要一次 LLM，在**节点内部**自建调用，不是 Registry 的第二趟 |
35//! | 意图分类 | 部分 | `before_generate` 内分类，结果写 `metadata`；**无**内置意图路由表 |
36//! | 意图执行（改 prompt/模型后交主模型） | 可 | 仍在前序 `before_generate` 中改 `ctx.request`，由步骤 3 一次生成 |
37//! | 意图执行（独立第二条 LLM 链） | 否（单条 pipeline） | 应用层多次 `generate`，或单节点内自建多轮调用 |
38//! | 输出整形 / 落库 | 是 | [`PipelineStage::after_generate`] |
39//!
40//! # 与字面线性链 A→B→C→D→输出的差异
41//!
42//! 若业务期望「全程线性、且 D 在模型之后」，应把 **D 放在 `after_generate`**；若某步必须在模型之前，应放在 **`before_generate`**，而不是假设存在「模型后的第四种全局钩子」。
43//!
44//! # 多步编排（多轮 Registry 调用）
45//!
46//! 若每个节点都要**各自**走 `ProviderRegistry::generate` 级别的一次往返，请在 Registry **之上**编排（多次调用或组合 pipeline），或在一个「复合」[`PipelineStage`] 内部封装多轮逻辑；本模块**不**替代通用工作流引擎。

52/// 单次调用在节点间传递的可变上下文。
53#[derive(Debug, Clone)]
54pub struct PipelineContext {
55    pub request: GenerateRequest,
56    pub response: Option<GenerateResponse>,
57}
58
59impl PipelineContext {
60    pub fn new(request: GenerateRequest) -> Self {
61        Self {
62            request,
63            response: None,
64        }
65    }
66
67    pub fn take_response(mut self) -> GenerateResponse {
68        self.response
69            .take()
70            .expect("pipeline after_generate must leave response set")
71    }
72}

74/// 一个流水线节点：默认钩子为空，只实现关心的阶段即可。
75pub trait PipelineStage: Send + Sync {
76    /// 稳定 id，用于日志 / 调试。
77    fn id(&self) -> &'static str;
78
79    /// 在调用 provider 之前；可改写 `ctx.request`。
80    fn before_generate(&self, _ctx: &mut PipelineContext) -> Result<(), LlmError> {
81        Ok(())
82    }
83
84    /// 在非流式 `generate` 得到响应之后；可改写 `ctx.response`。
85    fn after_generate(&self, _ctx: &mut PipelineContext) -> Result<(), LlmError> {
86        Ok(())
87    }
88
89    /// 在流式 `generate_stream` 完整结束之后调用；默认转调 [`PipelineStage::after_generate`]。
90    fn after_generate_stream(&self, ctx: &mut PipelineContext) -> Result<(), LlmError> {
91        self.after_generate(ctx)
92    }
93}

95/// 有序阶段列表；[`Pipeline::is_empty`] 或 registry 未设置 pipeline 时不跑任何钩子。
96#[derive(Clone, Default)]
97pub struct Pipeline {
98    stages: Vec<Arc<dyn PipelineStage>>,
99}
100
101impl Pipeline {
102    pub fn builder() -> PipelineBuilder {
103        PipelineBuilder::default()
104    }
105
106    pub fn is_empty(&self) -> bool {
107        self.stages.is_empty()
108    }
109
110    pub(crate) fn run_before(&self, ctx: &mut PipelineContext) -> Result<(), LlmError> {
111        for stage in &self.stages {
112            stage.before_generate(ctx)?;
113        }
114        Ok(())
115    }
116
117    pub(crate) fn run_after_generate(&self, ctx: &mut PipelineContext) -> Result<(), LlmError> {
118        for stage in &self.stages {
119            stage.after_generate(ctx)?;
120        }
121        Ok(())
122    }
123
124    pub(crate) fn run_after_stream(&self, ctx: &mut PipelineContext) -> Result<(), LlmError> {
125        for stage in &self.stages {
126            stage.after_generate_stream(ctx)?;
127        }
128        Ok(())
129    }
130}

132#[derive(Default)]
133pub struct PipelineBuilder {
134    stages: Vec<Arc<dyn PipelineStage>>,
135}
136
137impl PipelineBuilder {
138    pub fn push<S: PipelineStage + 'static>(mut self, stage: S) -> Self {
139        self.stages.push(Arc::new(stage));
140        self
141    }
142
143    pub fn push_arc(mut self, stage: Arc<dyn PipelineStage>) -> Self {
144        self.stages.push(stage);
145        self
146    }
147
148    pub fn build(self) -> Pipeline {
149        Pipeline {
150            stages: self.stages,
151        }
152    }
